

You are working on my Blood Donation web app (Angular frontend). I want a **premium, modern, mindblowing loading screen** that shows before the landing page appears, using the concept:

### Loader concept: **“Blood drop fill + shine sweep”**

A centered blood drop icon that:

1. starts as a clean outline
2. fills from bottom to top with a premium red gradient (liquid fill feel)
3. a soft diagonal “shine sweep” passes across the drop while filling
4. after fill completes, the loader gently fades out and reveals the app

This must feel like a **high-end SaaS + medical UI**, not childish. Smooth, minimal, elegant.

---

# Functional Requirements

1. Loader should show when app is accessed and Angular is bootstrapping:

* appears instantly on first load (no flash of unstyled page)
* stays until Angular renders the first route (landing page or any route)
* hides with a smooth fade/scale transition

2. Loader must be global:

* should work for all roles and routes
* should appear on initial app load
* OPTIONAL (nice-to-have): also show during route changes and major API fetches, but do not block usability for small actions

3. Theme compatible:

* must support both Light and Dark mode (we are adding theme toggle in the app)
* loader background and colors must adapt using CSS variables

4. Performance:

* pure HTML/CSS animation preferred (no heavy libraries)
* keep animation duration ~1.2s–1.8s loop if needed, but ideally a single satisfying sequence
* avoid excessive GPU-heavy blur; keep it smooth

---

# UX Specs (Premium / Sleek)

* Background: uses theme variable (light: near-white neutral, dark: deep neutral)
* Drop outline: subtle, crisp, slightly translucent border (theme-aware)
* Fill: vertical fill from bottom to top using a gradient:

  * deep red at bottom → slightly lighter red at top
* Shine sweep: soft diagonal highlight moving across the drop while filling:

  * like a glass reflection
  * low opacity, not flashy
* Text (optional): below the drop, minimal microcopy:

  * “Loading LifeLink…” OR “Preparing to save lives…”
  * use muted text color token
* Transition out:

  * fade out opacity 1 → 0 and slight scale 1 → 0.98
  * duration 250–400ms, ease-in-out
* Prevent scrolling while loader is visible

# Implementation Plan (Angular)

## A) Create Global Loader Overlay

1. Create a `LoaderService` (singleton) with:

* BehaviorSubject<boolean> `loading$`
* `show()`, `hide()`
* `initAppLoader()` to handle initial boot loader logic

2. Create `AppLoaderComponent`:

* full-screen overlay (position fixed, inset 0, high z-index)
* contains drop SVG + fill + shine layers
* uses CSS variables and works in light/dark

3. Add `<app-loader></app-loader>` at top-level (AppComponent template) so it exists on every route.

## B) Ensure it shows BEFORE Angular renders

Implement one of these robust approaches (choose best for this codebase):

### Option 1 (recommended): Static index.html Preloader + Angular takes over

* Add the loader markup directly in `src/index.html` as initial preloader
* When Angular bootstraps, it removes it smoothly
  Steps:
* Put `<div id="initial-loader">...</div>` in index.html
* In `main.ts` or `AppComponent.ngAfterViewInit()`, fade it out and remove from DOM
* This avoids “white flash” while Angular loads

### Option 2: Angular-only loader (acceptable)

* Show loader immediately via AppComponent
* But ensure critical styles are in global styles to reduce flash risk

**Use Option 1** unless there’s a strong reason not to.

# Loader Visual Implementation Details (Exact Animation)

## Drop Graphic

Use an inline SVG drop shape for crisp scaling.

* One path for outline
* One masked rect for fill
* One shine overlay layer

## Fill Animation

* Use a fill element masked by drop shape.
* Animate translateY from 100% → 0% to simulate filling upward.
* Add slight “liquid easing” (but no cartoon waves).
* Duration: ~1200ms

## Shine Sweep Animation

* Add a diagonal translucent gradient strip
* Animate from left-bottom to right-top across the drop
* Sync with fill (starts at ~20% progress, ends by 90%)
* Duration: ~900–1200ms

## Looping Behavior

* For initial load, play once, then stop and fade out when app ready.
* If route-change loader is enabled later, allow looping:

  * fill resets and repeats smoothly, but keep it calm.
